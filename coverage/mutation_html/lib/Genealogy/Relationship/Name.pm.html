  357: =head1 METHODS
  358: 
  359: =head2 name
  360: 
  361: Returns the name of the relationship between person A and person B.
  362: 
  363: =head3 PURPOSE
  364: 
  365: Given the number of steps from person A up to the nearest common ancestor
  366: (C<steps_to_ancestor>) and the number of steps from that ancestor down to
  367: person B (C<steps_from_ancestor>), plus the sex of person B and a language
  368: code, returns a localised relationship-name string.
  369: 
  370: =head3 ARGUMENTS
  371: 
  372: =over 4
  373: 
  374: =item C<steps_to_ancestor> (integer, required)
  375: 
  376: Number of generational steps from person A up to the common ancestor.
  377: Must be a non-negative integer.  Zero means person A I<is> the ancestor.
  378: 
  379: =item C<steps_from_ancestor> (integer, required)
  380: 
  381: Number of generational steps from the common ancestor down to person B.
  382: Must be a non-negative integer.
  383: 
  384: =item C<sex> (string, required)
  385: 
  386: Sex of person B.  Must be C<'M'> (male) or C<'F'> (female).
  387: 
  388: =item C<language> (string, optional)
  389: 
  390: BCP-47-style language tag (only the primary subtag is used).
  391: Supported values: C<en> (default), C<fr>, C<de>.
  392: 
  393: =back
  394: 
  395: =head3 RETURNS
  396: 
  397: A string containing the relationship name, or C<undef> if the combination
  398: is not found in the lookup table.
  399: 
  400: =head3 EXAMPLE
  401: 
  402:     my $namer = Genealogy::Relationship::Name->new();
  403: 
  404:     # Person A is the grandparent (2 steps up) of the common ancestor,
  405:     # and person B is 3 steps below the ancestor; B is female => first cousin once-removed
  406:     my $rel = $namer->name(
  407:         steps_to_ancestor   => 2,
  408:         steps_from_ancestor => 3,
  409:         sex                 => 'F',
  410:     );
  411: 
  412: =head3 API SPECIFICATION
  413: 
  414: =head4 Input
  415: 
  416:     {
  417: 	steps_to_ancestor   => { type => 'integer', minimum => 0 },
  418: 	steps_from_ancestor => { type => 'integer', minimum => 0 },
  419: 	sex                 => { type => 'string', memberof => ['M', 'F'] },
  420: 	language            => { type => 'string', regex => qr/^(?:en|de|fr)/, optional => 1 },
  421:     }
  422: 
  423: =head4 Output
  424: 
  425:     {
  426:         type     => 'string',
  427:         optional => 1,     # undef when the combination is not tabulated
  428:     }
  429: 
  430: =head3 FORMAL SPECIFICATION
  431: 
  432:     name ______________________________________________________
  433:     [In]  steps_to_ancestor   : N0
  434:           steps_from_ancestor : N0
  435:           sex                 : {M, F}

  605:

  607: language for all subsequent calls to C<name()>:

  612: The object is also compatible with C<Object::Configure> for runtime
  613: reconfiguration.
