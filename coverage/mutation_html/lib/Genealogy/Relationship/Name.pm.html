  320: =head1 DESCRIPTION
  321: 
  322: C<Genealogy::Relationship::Name> maps a pair of step-counts (person A to common
  323: ancestor, common ancestor to person B) plus the sex of person B and an optional
  324: language code to a human-readable relationship name string.
  325: 
  326: The relationship tables were originally embedded in the C<gedcom> and C<ged2site>
  327: distributions inside C<Gedcom::Individual::relationship_up()>; this module
  328: extracts them into a reusable, installable CPAN distribution.
  329: 
  330: Supported languages: C<en> (English, default), C<fr> (French), C<de> (German).

  433: =head2 name
  434: 
  435: Returns the name of the relationship between person A and person B.
  436: 
  437: =head3 PURPOSE
  438: 
  439: Given the number of steps from person A up to the nearest common ancestor
  440: (C<steps_to_ancestor>) and the number of steps from that ancestor down to
  441: person B (C<steps_from_ancestor>), plus the sex of person B and a language
  442: code, returns a localised relationship-name string.
  443: 
  444: =head3 ARGUMENTS
  445: 
  446: =over 4
  447: 
  448: =item C<steps_to_ancestor> (integer, required)
  449: 
  450: Number of generational steps from person A up to the common ancestor.
  451: Must be a non-negative integer.  Zero means person A I<is> the ancestor.
  452: 
  453: =item C<steps_from_ancestor> (integer, required)
  454: 
  455: Number of generational steps from the common ancestor down to person B.
  456: Must be a non-negative integer.
  457: 
  458: =item C<sex> (string, required)
  459: 
  460: Sex of person B.  Must be C<'M'> (male) or C<'F'> (female).
  461: 
  462: =item C<language> (string, optional)
  463: 
  464: BCP-47-style language tag (only the primary subtag is used).
  465: Supported values: C<en> (default), C<fr>, C<de>.
  466: 
  467: =item C<person> (object, optional)
  468: 
  469: An optional person object (e.g. a C<Gedcom::Individual> instance) passed
  470: through to the error handler when an error occurs.  Takes priority over the
  471: C<ctx> set at construction time.  The handler receives it as C<ctx> (logger
  472: path) or C<person> (on_error path), matching the C<complain()> interface
  473: in C<gedcom>/C<ged2site>.
  474: 
  475: =back
  476: 
  477: =head3 RETURNS
  478: 
  479: A string containing the relationship name, or C<undef> if the combination
  480: is not found in the lookup table.
  481: 
  482: =head3 EXAMPLE
  483: 
  484:     my $namer = Genealogy::Relationship::Name->new();

  486:     # Person A is the grandparent (2 steps up) of the common ancestor,
  487:     # and person B is 3 steps below the ancestor; B is female => first cousin once-removed

  491:         sex                 => 'F',
  492:     );

  507:

  702: =head1 CONFIGURATION
  703: 
  704: The constructor accepts an optional C<language> key which sets the default
  705: language for all subsequent calls to C<name()>:
  706: 
  707:     my $namer = Genealogy::Relationship::Name->new(language => 'fr');
  708: 
  709: This default can be overridden per-call by passing C<language> to C<name()>.
  710: The object is also compatible with C<Object::Configure> for runtime
  711: reconfiguration.
  712: 
  713: =head2 Error handling
  714: 
  715: Errors are dispatched through the following priority chain:
  716: 
  717: =over 4
  718: 
  719: =item 1. C<logger> coderef (L<Log::Abstraction> E<gt>= 0.28)
  720: 
  721: The preferred integration path for C<gedcom>/C<ged2site>.  Pass a C<logger>
  722: coderef and a C<ctx> value (typically a C<Gedcom::Individual> object) to
  723: C<new()>.  On error, C<_error()> calls the logger with a hashref containing
  724: C<class>, C<file>, C<line>, C<level> (C<"error">), C<message>, and C<ctx>.
  725: This matches the shape that L<Log::Abstraction> 0.28 uses for all coderef
  726: logger callbacks, so the same handler works for both normal logging and errors.
  727: 
  728:     my $namer = Genealogy::Relationship::Name->new(
  729:         logger => sub {
  730:             my $args = shift;
  731:             complain(
  732:                 warning => join('', @{$args->{message}}),
  733:                 person  => $args->{ctx},
  734:             );
  735:         },
  736:         ctx => $individual,   # Gedcom::Individual for this lookup
  737:     );
  738: 
  739: =item 2. C<on_error> coderef
  740: 
  741: A simpler fallback for callers not using L<Log::Abstraction>.  The coderef
  742: is called with a flat hash matching the C<complain()> interface in
  743: C<gedcom>/C<ged2site>: C<warning =E<gt> $msg, person =E<gt> $person>.
  744: C<person> is resolved from the per-call C<name()> argument first, then from
  745: the constructor C<ctx>, so a single handler works in both cases.
  746: 
  747:     my $namer = Genealogy::Relationship::Name->new(
  748:         on_error => sub {
  749:             my %args = @_;
  750:             complain(warning => $args{warning}, person => $args{person});
  751:         },
  752:     );
  753: 
  754: =item 3. C<croak> (default)
  755: 
  756: When neither C<logger> nor C<on_error> is set, errors croak normally.
  757: 
  758: =back
  759: 
  760: If any handler returns without dying (e.g. a warning-only handler),
  761: C<name()> returns C<undef>.
  762: 
  763: =head2 ctx
  764: 
  765: The C<ctx> constructor argument is stored on the object and passed to the
  766: C<logger> coderef (as C<$args-E<gt>{ctx}>) and to the C<on_error> coderef
  767: (as C<person =E<gt> $ctx>) on every error.  A per-call C<person> argument
  768: to C<name()> takes priority over C<ctx> when both are present.
  769: 
  770: =head1 DIAGNOSTICS
  771: 
  772: =over 4
  773: 
  774: =item on_error must be a CODE reference
  775: 
  776: C<on_error> was passed to C<new()> but is not a code reference.
  777: 
  778: =item steps_to_ancestor must be a defined non-negative integer
  779: 
  780: C<steps_to_ancestor> was C<undef>; this is not caught by C<validate_strict>
  781: for C<type =E<gt> 'integer'> and is caught explicitly.
  782: 
  783: =item steps_from_ancestor must be a defined non-negative integer
  784: 
  785: As above, for C<steps_from_ancestor>.

  787: =back
