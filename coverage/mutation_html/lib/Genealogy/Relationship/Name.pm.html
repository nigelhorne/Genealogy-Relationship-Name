  483: =head2 name
  484: 
  485: Returns the name of the relationship between person A and person B.
  486: 
  487: =head3 PURPOSE
  488: 
  489: Given the number of steps from person A up to the nearest common ancestor
  490: (C<steps_to_ancestor>) and the number of steps from that ancestor down to
  491: person B (C<steps_from_ancestor>), plus the sex of person B and a language
  492: code, returns a localised relationship-name string.

  495:

  502:

  504:

  513:

  747: =head1 CONFIGURATION
  748: 
  749: The constructor accepts an optional C<language> key which sets the default
  750: language for all subsequent calls to C<name()>:
  751: 
  752:     my $namer = Genealogy::Relationship::Name->new(language => 'fr');
  753: 
  754: This default can be overridden per-call by passing C<language> to C<name()>.
  755: The object is also compatible with C<Object::Configure> for runtime
  756: reconfiguration.
  757: 
  758: =head2 Error handling
  759: 
  760: Errors are dispatched through the following priority chain:
  761: 
  762: =over 4
  763: 
  764: =item 1. L<Log::Abstraction> object (preferred)
  765: 
  766: Construct a C<Log::Abstraction> object with the desired logger coderef and
  767: C<ctx> (typically a C<Gedcom::Individual>), then pass it as C<logger> to
  768: C<new()>.  On error, this module simply calls C<< $logger->error($msg) >>
  769: and Log::Abstraction handles ctx forwarding, formatting, and dispatch.
  770: 
  771:     use Log::Abstraction;
  772: 
  773:     my $logger = Log::Abstraction->new(
  774:         logger => sub {
  775:             my $args = shift;
  776:             my $msg = $args->{ctx}
  777:                 ? $args->{ctx}->as_string() . ': ' . join('', @{$args->{message}})
  778:                 : join('', @{$args->{message}});
  779:             complain({ message => $msg, person => $args->{ctx} });
  780:         },
  781:         ctx => $individual,
  782:     );
  783: 
  784:     my $namer = Genealogy::Relationship::Name->new(logger => $logger);
  785: 
  786: =back
  787: 
  788: If any handler returns without dying (e.g. a warning-only handler in
  789: L<Log::Abstraction>), C<name()> returns C<undef>.
  790: 
  791: =head1 DIAGNOSTICS

  798: distinct from omitting the argument; use a defined non-negative integer.
